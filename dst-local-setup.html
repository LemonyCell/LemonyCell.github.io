Don't Starve Together
Локальна мережа

Мінімальна затримка — втрьох через WiFi hotspot

Мета

Що хочемо досягти

Зіграти втрьох у DST через локальну мережу з мінімальною затримкою — без зовнішніх серверів, весь трафік локально.

Контекст

Обмеження та обставини

  • Хост-ноутбук (Windows) роздає WiFi-точку доступу і запускає сервер DST одночасно
  • Для створення hotspot на Windows потрібне підключення до роутера через WiFi — кабелю немає
  • Хост-ноутбук одночасно підключений до роутера через WiFi і роздає власну точку доступу
  • Один ноутбук працює на WinToGo — до роутера не підключається, але до hotspot може
  • Один ноутбук — Mac
  • При підключенні до hotspot без інтернету Steam пропонує «Play Offline»

Спроби

Що вже зробили

  • Хост створив WiFi hotspot, інші підключились
  • Шукали сервер через LAN browse у DST — не знайшли
  • Firewall на Windows не налаштовували
  • На Mac і WinToGo нічого не робили

Рекомендації

Що робити

1. Firewall — головна причина проблеми

Windows класифікує hotspot-мережу як Public — більшість правил firewall працюють лише для Private. Потрібно додати правило вручну:

Firewall → Advanced Settings
→ Inbound Rules → New Rule
→ Port → UDP → 10999
→ Allow
→ Увімкнути всі три профілі:
  Domain  ✓  Private  ✓  Public  ✓

2. Якщо LAN browse досі не показує сервер

Підключитись напряму по IP. На хост-ноутбуку відкрити CMD і виконати ipconfig. Знайти IP hotspot-адаптера (зазвичай 192.168.137.1). Потім у DST натиснути ~ та ввести:

c_connect("192.168.137.1", 10999)

3. Mac — нічого не потрібно

Firewall на Mac вимкнений за замовчуванням. Як клієнт — жодних налаштувань не потребує.

4. WinToGo — якщо не з'єднується

Запасний варіант: USB-тетеринг з телефону або USB-Ethernet адаптер.

5. «Play Offline» у Steam

Це нормально. Натиснути Play Offline на всіх ноутбуках — LAN-гра при цьому працює повноцінно.
